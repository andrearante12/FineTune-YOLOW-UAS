## Project Overview

Fine-tuned YOLO-World open-vocabulary object detection model on the VisDrone dataset for real-time UAS (drone) applications. Forked from [AILab-CVC/YOLO-World](https://github.com/AILab-CVC/YOLO-World). Uses both the Ultralytics API (simple path) and the full MMDetection ecosystem (advanced path).

## Environment Setup

```bash
conda env create -f yoloworld_stable.yml
conda activate yoloworld_stable
pip install -e .
```

Python 3.10, PyTorch 2.12.0, CUDA 12.8. Key deps: mmengine, mmdet==3.0.0, mmyolo, transformers, supervision==0.19.0.

## Common Commands

### Training
```bash
# Simple fine-tuning via Ultralytics API
python scripts/finetune.py

# Advanced training via MMDetection
PYTHONPATH=./ python tools/train.py configs/visdrone/yolo_world_v2_x_visdrone.py
```

### Inference
```bash
# Video inference (edit scripts/run_inference.py to change target video/classes)
python scripts/run_inference.py

# Jetson-optimized real-time inference
python scripts/yolo_world_jetson.py

# Demo with MMDetection
PYTHONPATH=./ python demo/image_demo.py <config> <weights> <image_dir> 'class1,class2'
```

# Convert VisDrone YOLO-format labels to COCO format

## Architecture

**Dual-path design:** The project supports two training/inference APIs:

1. **Ultralytics API** (`scripts/finetune.py`, `scripts/run_inference.py`): Simple wrapper using `ultralytics.YOLOWorld`. Good for quick fine-tuning and inference.
2. **MMDetection API** (`tools/train.py`, `tools/test.py`): Full config-driven pipeline via mmengine/mmdet/mmyolo. Used for advanced training with the configs in `configs/`.

**Core model** (`yolo_world/models/`):
- `detectors/yolo_world.py` — YOLOWorldDetector: extends YOLODetector with multi-modal support. Key method: `set_classes()` for dynamic class vocabulary.
- `backbones/` — Dual encoders: vision encoder (image features) + text encoder (CLIP-based, typically frozen during fine-tuning).
- `necks/` — Feature pyramid networks for multi-scale detection.
- `dense_heads/` — YOLOWorldHead with text-image feature alignment.

**Config structure** (`configs/`):
- `configs/visdrone/` — VisDrone fine-tuning configs (primary use case).
- Other dirs cover COCO fine-tuning, prompt tuning, pre-training, and segmentation.

**Custom scripts** (`scripts/`): All project-specific scripts for training, evaluation, data processing, and deployment. `sweep_checkpoints.py` imports from `evaluate.py` and `evaluate_open_vocab.py` as siblings.

**Dataset pipeline** (`yolo_world/datasets/`): Multi-modal datasets wrapping YOLOv5-style COCO/LVIS loaders with text prompt injection via `class_text_path`.

## Critical Design Decisions

- **CLIP text encoder must stay frozen** during fine-tuning. Unfreezing it causes the model to overfit to VisDrone classes and lose open-vocabulary capability. In `scripts/finetune.py`, parameters with `"txt_model"` in their name have `requires_grad=False`.
- **Short training is intentional.** Training beyond ~5 epochs degrades open-vocab performance. The 80-epoch config in `scripts/finetune.py` exists for experimentation but 5 epochs showed best balance (see `docs/RESULTS.md`).
- **Descriptive prompts** like `"foreign object on driveway"` are used to catch objects not in any predefined class list. This is a core feature of the open-vocab approach.

## Deployment

- **Jetson**: `scripts/yolo_world_jetson.py` with FP16 and real-time FPS display
- **Docker**: `docker build -t yolo_demo .` (CUDA 11.8 + Gradio UI)
- **ONNX**: Via `deploy/easydeploy/` module
- Training was done on 2x RTX 5090 (32GB) + Threadripper PRO
